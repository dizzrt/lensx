## Purpose

定义 lensX 启动器面板的布局、默认状态、搜索状态、插件打开态、插件 action 搜索、内容高度自适应、原生窗口自动高度、输入与拖拽兼容，以及前端基座接入要求。
## Requirements
### Requirement: 启动器必须渲染一体化面板布局

启动器主界面 MUST 渲染为一体化面板，顶部为状态区域，下方为内容主体；默认状态和搜索状态下顶部状态区域 MUST 展示搜索输入区，插件页面打开状态下顶部状态区域 MUST 展示插件标题区。顶部状态区域和内容主体 MUST 共用同一卡片视觉容器，并避免使用明显的分隔线表达二者边界。

#### Scenario: 默认渲染一体化面板

- **WHEN** 启动器窗口打开且搜索输入为空
- **THEN** 界面展示顶部搜索输入区和下方内容主体
- **THEN** 搜索输入区与内容主体在视觉上属于同一张卡片

#### Scenario: 插件打开时渲染一体化面板

- **WHEN** 启动器打开某个插件页面
- **THEN** 界面展示顶部插件标题区和下方插件内容主体
- **THEN** 插件标题区与插件内容主体在视觉上属于同一张卡片

#### Scenario: 不展示被排除入口

- **WHEN** 启动器窗口展示默认状态、搜索状态或插件页面打开状态
- **THEN** 界面不展示“市场精选”分区
- **THEN** 顶部状态区域右侧不展示头像栏或账号入口

### Requirement: 插件打开态顶部必须展示插件名称和关闭按钮

当启动器存在当前插件页面时，顶部状态区域 MUST 从搜索输入区切换为插件标题区。插件标题区 MUST 只展示当前插件名称和关闭按钮，MUST NOT 展示当前页面名称。关闭按钮 MUST 紧跟插件名称后方展示，MUST NOT 放置到顶部状态区域最右侧。

#### Scenario: 打开插件后顶部展示插件名称

- **WHEN** 用户通过插件 action 打开某个插件页面
- **THEN** 顶部状态区域不展示搜索输入框
- **THEN** 顶部状态区域展示当前插件名称
- **THEN** 顶部状态区域不展示当前页面名称

#### Scenario: 关闭按钮紧跟插件名称

- **WHEN** 顶部状态区域处于插件打开态
- **THEN** 关闭按钮展示在插件名称右侧相邻位置
- **THEN** 关闭按钮不靠顶部状态区域最右侧对齐
- **THEN** 插件名称较长时关闭按钮仍保持可见

#### Scenario: 点击关闭按钮回到工具主页

- **WHEN** 顶部状态区域处于插件打开态
- **AND** 用户点击关闭按钮
- **THEN** 系统关闭当前插件页面
- **THEN** 系统清空搜索词
- **THEN** 内容主体恢复展示工具主页默认内容
- **THEN** 顶部状态区域恢复展示搜索输入区

#### Scenario: 插件标题区接入前端基座

- **WHEN** 顶部状态区域展示插件标题区
- **THEN** 关闭按钮的可访问名称等用户可见文案来自应用级 i18n message
- **THEN** 插件标题区位于现有 Naive UI Provider、应用主题、Naive UI locale/dateLocale 同步机制之下
- **THEN** 插件标题区在 light 与 dark 主题下保持可读

### Requirement: 默认状态必须展示最近使用和已固定

搜索输入为空时，启动器内容主体 MUST 展示“最近使用”和“已固定”两个分区；本变更中的条目数据 MUST 作为前端表现层 mock 数据处理，不要求接入真实数据源。搜索输入为空时，启动器内容主体 MUST NOT 展示已注册插件列表、插件 registry 入口分区或插件 action 直出列表。

#### Scenario: 搜索为空时展示默认分区

- **WHEN** 用户打开启动器且搜索输入为空
- **THEN** 内容主体展示“最近使用”分区
- **THEN** 内容主体展示“已固定”分区
- **THEN** 内容主体不展示已注册插件列表
- **THEN** 内容主体不展示插件 registry 入口分区

#### Scenario: 默认条目来自表现层数据

- **WHEN** 默认分区展示条目
- **THEN** 条目可用于验证视觉布局和交互状态
- **THEN** 系统不要求读取真实最近使用记录或固定项持久化数据

#### Scenario: 已固定设置条目保持视觉 mock

- **WHEN** 搜索输入为空
- **AND** “已固定”分区展示设置相关条目
- **THEN** 该条目作为表现层 mock 数据展示
- **THEN** 系统不要求该条目打开设置插件页面

#### Scenario: 插件 registry 不作为默认态入口展示

- **WHEN** 应用已加载插件 registry
- **AND** 用户打开启动器且搜索输入为空
- **THEN** 内容主体仍只展示“最近使用”和“已固定”两个分区
- **THEN** 系统 MUST NOT 因 registry 中存在插件 actions 而在默认态追加插件分区

### Requirement: 搜索状态必须仅展示匹配结果

搜索输入存在内容时，启动器内容主体 MUST 切换为搜索状态，并且 MUST 仅展示“匹配结果”分区；插件 action 的真实搜索行为由插件搜索相关要求约束，本要求仅约束搜索状态的分区切换。

#### Scenario: 输入搜索词后切换到匹配结果

- **WHEN** 用户在搜索输入区输入任意非空内容
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 内容主体不展示“最近使用”和“已固定”分区

#### Scenario: 清空搜索词后恢复默认状态

- **WHEN** 用户清空搜索输入区内容
- **THEN** 内容主体恢复展示“最近使用”和“已固定”分区

### Requirement: 搜索栏必须搜索已安装插件行为

启动器搜索栏 MUST 作为 launcher 固有内部入口搜索已安装插件暴露的 actions。系统 MUST 从插件 registry 中读取可用 actions，并将每个搜索结果映射到可执行的插件 action。搜索匹配范围 MUST 包含插件 action 标题、插件 action ID、插件 ID、插件英文名、当前应用语言下的插件名称以及该插件所有有效别名。每个结果的可见主标题 MUST 展示所属插件在当前应用语言下的名称；当前语言名称缺失时 MUST 回退英文名。系统 MUST NOT 将插件 action 标题作为搜索结果的可见主标题。搜索能力 MUST NOT 作为内建搜索插件注册。

#### Scenario: 输入插件 action 标题后展示结果

- **WHEN** 应用已加载插件 registry
- **AND** 用户在启动器搜索栏输入与某个插件 action 标题匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件 action
- **THEN** 结果的可见主标题展示该 action 所属插件的当前语言名称，而不展示该 action 标题

#### Scenario: 输入插件英文名后展示所属 action

- **WHEN** 应用已加载插件 registry
- **AND** 用户在启动器搜索栏输入与某个插件英文名匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件暴露的可打开 actions

#### Scenario: 输入当前语言插件名称后展示所属 action

- **WHEN** 应用已加载插件 registry
- **AND** 应用 locale 为 `zh-CN`
- **AND** 用户在启动器搜索栏输入与某个插件 `zh-CN` 名称匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件暴露的可打开 actions
- **THEN** 每个结果的可见主标题展示该插件的 `zh-CN` 名称

#### Scenario: 当前语言插件名称缺失时搜索英文名

- **WHEN** 应用已加载插件 registry
- **AND** 应用 locale 对应的插件名称缺失
- **AND** 用户在启动器搜索栏输入与该插件英文名匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件暴露的可打开 actions
- **THEN** 每个结果的可见主标题展示该插件的英文名

#### Scenario: 输入插件有效别名后展示所属 action

- **WHEN** 应用已加载插件 registry
- **AND** 用户在启动器搜索栏输入与某个插件有效别名匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件暴露的可打开 actions

#### Scenario: 已禁用默认别名不参与搜索

- **WHEN** 应用已加载插件 registry
- **AND** 用户已在设置中删除某个插件默认别名
- **AND** 用户在启动器搜索栏输入该已删除默认别名
- **THEN** 系统不再因为该默认别名命中该插件 action

#### Scenario: 重新启用默认别名后参与搜索

- **WHEN** 应用已加载插件 registry
- **AND** 用户重新添加此前删除的默认别名
- **AND** 用户在启动器搜索栏输入该默认别名
- **THEN** 匹配结果包含该插件暴露的可打开 actions

#### Scenario: 输入插件 ID 后展示所属 action

- **WHEN** 应用已加载插件 registry
- **AND** 用户在启动器搜索栏输入与某个插件 ID 匹配的关键词
- **THEN** 内容主体展示“匹配结果”分区
- **THEN** 匹配结果包含该插件暴露的可打开 actions

#### Scenario: 点击搜索结果打开插件页面

- **WHEN** 用户点击某个插件 action 搜索结果
- **THEN** 系统通过现有插件 action dispatcher 触发该 action
- **THEN** 插件页面出口打开该 action 的目标页面
- **THEN** 插件页面独占搜索框下方的 launcher 主体区域

#### Scenario: 搜索无结果时展示空状态

- **WHEN** 应用已加载插件 registry
- **AND** 用户输入的关键词无法匹配任何插件 action、插件 ID、插件英文名、当前语言插件名称或有效别名
- **THEN** 内容主体展示搜索空状态
- **THEN** 系统 MUST NOT 展示表现层 mock 搜索结果

#### Scenario: 插件 registry 加载失败时展示错误

- **WHEN** 用户输入搜索词
- **AND** 插件 registry 加载失败
- **THEN** 内容主体展示可诊断的插件 registry 错误
- **THEN** 系统 MUST NOT 使用假数据伪装真实搜索结果

### Requirement: 插件搜索结果必须接入前端基座

插件搜索结果的用户可见文案 MUST 来自应用 i18n message。搜索结果 UI MUST 位于现有 Naive UI Provider、应用主题、Naive UI locale/dateLocale 同步机制之下，并 MUST 兼容 light 和 dark 主题。静态布局 MUST 优先使用 UnoCSS；语义样式、状态样式和主题变量桥接 MUST 使用 Less。

#### Scenario: 搜索结果文案来自应用 i18n

- **WHEN** 启动器展示插件搜索结果、空状态、错误状态或操作文案
- **THEN** 这些文案来自应用级 i18n message

#### Scenario: 搜索结果兼容明暗主题

- **WHEN** 应用主题在 light 与 dark 之间切换
- **THEN** 插件搜索结果的文本、背景、边框、hover 状态和 focus 状态保持可读
- **THEN** 页面不出现亮色硬编码导致的暗色主题冲突

### Requirement: 内容主体高度必须自适应并限制最大高度

启动器内容主体 MUST 根据内容自然高度变化；当内容高度超过主体最大高度时，原生窗口 MUST 停止继续增高，并且内容主体内部 MUST 展示垂直滚动能力。原生窗口最大高度 MUST 为 `800px`。

#### Scenario: 内容较少时主体自然收缩

- **WHEN** 内容主体只有少量条目
- **THEN** 内容主体高度贴合内容
- **THEN** 启动器面板不会保留明显多余空白

#### Scenario: 内容超过最大高度时内部滚动

- **WHEN** 内容主体条目数量超过主体最大高度可容纳范围
- **THEN** 内容主体高度不超过最大高度
- **THEN** 用户可以在内容主体内部垂直滚动查看更多条目
- **THEN** 搜索输入区保持在滚动区域之外

### Requirement: 原生窗口高度必须随面板内容自动变化

Tauri 主窗口高度 MUST 根据启动器面板内容高度自动调整，并 MUST 采用顶部位置固定、向下展开或收起的策略；窗口宽度 MUST 保持稳定，窗口位置 MUST 不因高度变化而主动重新居中。

#### Scenario: 内容变多时窗口向下展开

- **WHEN** 面板内容高度增加且未超过最大窗口高度
- **THEN** 原生窗口高度增加以容纳面板内容
- **THEN** 原生窗口顶部位置保持不变

#### Scenario: 内容变少时窗口向上收起到底部

- **WHEN** 面板内容高度减少
- **THEN** 原生窗口高度减少以贴合面板内容
- **THEN** 原生窗口顶部位置保持不变

#### Scenario: 内容超过最大窗口高度时窗口停止增高

- **WHEN** 面板内容高度超过最大窗口高度
- **THEN** 原生窗口高度不超过 `800px`
- **THEN** 超出内容通过内容主体内部滚动查看

#### Scenario: 窗口宽度保持固定

- **WHEN** 面板内容高度发生变化
- **THEN** 原生窗口宽度保持为 `650px`
- **THEN** 原生窗口仅根据内容调整高度

### Requirement: 搜索输入和窗口拖拽必须兼容

搜索输入控件 MUST 支持点击聚焦、文本输入和文本选择；输入控件之外的可拖拽区域 MUST 支持移动整个 Tauri 窗口。

#### Scenario: 用户点击搜索框输入内容

- **WHEN** 用户点击搜索输入控件
- **THEN** 搜索输入控件获得焦点
- **THEN** 用户可以输入和编辑搜索文本

#### Scenario: 用户拖拽非输入区域移动窗口

- **WHEN** 用户拖拽搜索输入控件之外的面板可拖拽区域
- **THEN** Tauri 窗口随拖拽移动

#### Scenario: 拖拽区域不破坏输入交互

- **WHEN** 用户在搜索输入控件内点击、定位光标或选择文本
- **THEN** 操作不会被窗口拖拽行为抢占

### Requirement: 面板文案和主题必须接入前端基座

启动器面板新增用户可见文案 MUST 通过应用级 i18n message 管理；面板样式 MUST 兼容 Naive UI 明暗主题，并 MUST 遵循 UnoCSS 与 Less 的样式分工。

#### Scenario: 面板文案来自应用 i18n

- **WHEN** 启动器面板展示搜索占位、分区标题或辅助操作文案
- **THEN** 这些文案来自应用级 i18n message

#### Scenario: 面板兼容明暗主题

- **WHEN** 应用主题在 light 与 dark 之间切换
- **THEN** 启动器面板背景、文本、边框、阴影和滚动区域保持可读且不出现亮色硬编码冲突

#### Scenario: 面板继续使用 Naive UI Provider

- **WHEN** 启动器面板渲染 Naive UI 组件
- **THEN** 组件位于现有 Naive UI Provider、locale 和 dateLocale 同步机制之下

### Requirement: 启动器面板必须支持桌面快捷键激活

启动器面板 MUST 能够由 Rust/Tauri 桌面快捷键动作可靠激活。通过快捷键显示窗口时，面板现有布局、搜索状态、内容高度自适应和拖拽兼容行为 MUST 保持有效。

#### Scenario: 快捷键显示面板后保持布局

- **WHEN** 用户通过桌面全局快捷键显示启动器窗口
- **THEN** 启动器面板渲染为现有一体化面板布局
- **THEN** 面板内容高度自适应和最大高度限制继续生效

#### Scenario: 快捷键显示面板后输入可交互

- **WHEN** 用户通过桌面全局快捷键显示启动器窗口
- **THEN** 搜索输入控件可点击、可输入、可编辑
- **THEN** 输入控件交互不被窗口拖拽行为抢占

### Requirement: 启动器面板失焦后必须自动隐藏

启动器面板 MUST 在主窗口失去焦点时自动隐藏，使用户点击其他应用或桌面区域后无需手动关闭面板。失焦隐藏 MUST 复用 Rust/Tauri 侧启动器窗口隐藏动作，并保持应用进程常驻。

#### Scenario: 点击其他位置后隐藏面板

- **WHEN** 启动器窗口当前可见
- **AND** 用户点击其他应用、桌面区域或其他可聚焦窗口
- **THEN** 启动器窗口被隐藏
- **THEN** 应用进程继续运行
