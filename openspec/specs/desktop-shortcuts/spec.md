## Purpose

定义 lensX 桌面全局快捷键的 Rust 侧管理、默认启动器激活快捷键、快捷键生命周期操作、冲突检测入口，以及启动器窗口动作路由要求。
## Requirements
### Requirement: 桌面快捷键必须由 Rust 侧管理器统一注册

系统 MUST 在 Rust 侧通过统一的快捷键管理器注册全局快捷键绑定，而不是在单个快捷键回调中直接写死窗口行为。每个绑定 MUST 至少包含稳定标识、快捷键组合、动作标识和启用状态。

#### Scenario: 应用启动时注册默认快捷键

- **WHEN** 应用启动并初始化桌面层
- **THEN** 系统通过快捷键管理器注册默认全局快捷键绑定
- **THEN** 默认绑定指向启动器窗口激活动作

#### Scenario: 禁用绑定不会注册快捷键

- **WHEN** 快捷键绑定的启用状态为禁用
- **THEN** 系统不向全局快捷键插件注册该绑定

### Requirement: 默认全局快捷键必须激活启动器窗口

系统 MUST 提供默认全局快捷键 `Ctrl+Shift+Space` 用于激活启动器窗口。触发默认快捷键时，系统 MUST 通过动作路由执行启动器窗口动作，而不是直接在快捷键 handler 中操作窗口。

#### Scenario: 触发默认快捷键激活启动器

- **WHEN** 用户按下 `Ctrl+Shift+Space`
- **THEN** 系统执行默认绑定对应的启动器窗口动作
- **THEN** 主窗口进入可交互的激活状态

### Requirement: 快捷键管理器必须支持注册生命周期操作

系统 MUST 在 Rust 侧提供注册、注销和重新注册快捷键绑定的生命周期操作。重新注册绑定时，系统 MUST 避免保留已经失效的旧绑定。

#### Scenario: 重新注册快捷键绑定

- **WHEN** 系统应用新的快捷键绑定集合
- **THEN** 系统注销不再启用或已变化的旧绑定
- **THEN** 系统注册新的启用绑定

#### Scenario: 注册失败返回错误

- **WHEN** 全局快捷键插件拒绝注册某个绑定
- **THEN** 系统返回可诊断的错误结果
- **THEN** 系统不因该错误直接崩溃

### Requirement: 快捷键管理器必须支持冲突检测入口

系统 MUST 提供检测快捷键是否已被注册或不可用的 Rust 侧入口，用于后续自定义快捷键能力接入。

#### Scenario: 检测已占用快捷键

- **WHEN** 待注册快捷键已经被当前应用注册
- **THEN** 系统报告该快捷键不可用

#### Scenario: 检测可用快捷键

- **WHEN** 待注册快捷键未被当前应用注册且全局快捷键插件允许注册
- **THEN** 系统报告该快捷键可用于绑定

### Requirement: 启动器快捷键动作必须覆盖显示隐藏切换

系统 MUST 在 Rust 侧提供启动器窗口动作路由，至少支持显示、隐藏和切换启动器主窗口。显示动作 MUST 尽量恢复窗口、展示窗口并请求焦点；隐藏动作 MUST 隐藏窗口但保持应用进程运行。

#### Scenario: 显示启动器窗口

- **WHEN** 系统执行 `launcher.show` 动作
- **THEN** 主窗口被展示
- **THEN** 系统请求主窗口获得焦点

#### Scenario: 隐藏启动器窗口

- **WHEN** 系统执行 `launcher.hide` 动作
- **THEN** 主窗口被隐藏
- **THEN** 应用进程继续运行

#### Scenario: 切换启动器窗口

- **WHEN** 系统执行 `launcher.toggle` 动作且主窗口当前可见
- **THEN** 主窗口被隐藏

#### Scenario: 从隐藏状态切换启动器窗口

- **WHEN** 系统执行 `launcher.toggle` 动作且主窗口当前不可见
- **THEN** 主窗口被展示
- **THEN** 系统请求主窗口获得焦点

### Requirement: 快捷键设置页必须只读展示桌面快捷键绑定

系统 MUST 支持设置插件以只读方式展示当前桌面快捷键绑定。该展示能力 MUST NOT 改变快捷键注册生命周期、默认快捷键、冲突检测或启动器窗口动作路由。

#### Scenario: 展示当前默认绑定

- **WHEN** 用户打开设置插件的快捷键页面
- **THEN** 页面展示默认启动器快捷键 `Ctrl+Shift+Space`
- **THEN** 页面展示该快捷键对应启动器窗口激活动作

#### Scenario: 展示不改变注册状态

- **WHEN** 快捷键设置页读取或展示快捷键绑定
- **THEN** 系统不重新注册快捷键
- **THEN** 系统不注销当前快捷键
- **THEN** 系统不改变默认快捷键动作路由

#### Scenario: 不提供编辑入口

- **WHEN** 用户查看快捷键设置页
- **THEN** 页面不提供录制快捷键、保存快捷键或重新绑定快捷键的操作
- **THEN** 系统不暴露用于本次设置页编辑快捷键的写入能力
