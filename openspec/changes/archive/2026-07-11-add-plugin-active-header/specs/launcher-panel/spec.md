## MODIFIED Requirements

### Requirement: 启动器必须渲染一体化面板布局

启动器主界面 MUST 渲染为一体化面板，顶部为状态区域，下方为内容主体；默认状态和搜索状态下顶部状态区域 MUST 展示搜索输入区，插件页面打开状态下顶部状态区域 MUST 展示插件标题区。顶部状态区域和内容主体 MUST 共用同一卡片视觉容器，并避免使用明显的分隔线表达二者边界。

#### Scenario: 默认渲染一体化面板

- **WHEN** 启动器窗口打开且搜索输入为空
- **THEN** 界面展示顶部搜索输入区和下方内容主体
- **THEN** 搜索输入区与内容主体在视觉上属于同一张卡片

#### Scenario: 插件打开时渲染一体化面板

- **WHEN** 启动器打开某个插件页面
- **THEN** 界面展示顶部插件标题区和下方插件内容主体
- **THEN** 插件标题区与插件内容主体在视觉上属于同一张卡片

#### Scenario: 不展示被排除入口

- **WHEN** 启动器窗口展示默认状态、搜索状态或插件页面打开状态
- **THEN** 界面不展示“市场精选”分区
- **THEN** 顶部状态区域右侧不展示头像栏或账号入口

## ADDED Requirements

### Requirement: 插件打开态顶部必须展示插件名称和关闭按钮

当启动器存在当前插件页面时，顶部状态区域 MUST 从搜索输入区切换为插件标题区。插件标题区 MUST 只展示当前插件名称和关闭按钮，MUST NOT 展示当前页面名称。关闭按钮 MUST 紧跟插件名称后方展示，MUST NOT 放置到顶部状态区域最右侧。

#### Scenario: 打开插件后顶部展示插件名称

- **WHEN** 用户通过插件 action 打开某个插件页面
- **THEN** 顶部状态区域不展示搜索输入框
- **THEN** 顶部状态区域展示当前插件名称
- **THEN** 顶部状态区域不展示当前页面名称

#### Scenario: 关闭按钮紧跟插件名称

- **WHEN** 顶部状态区域处于插件打开态
- **THEN** 关闭按钮展示在插件名称右侧相邻位置
- **THEN** 关闭按钮不靠顶部状态区域最右侧对齐
- **THEN** 插件名称较长时关闭按钮仍保持可见

#### Scenario: 点击关闭按钮回到工具主页

- **WHEN** 顶部状态区域处于插件打开态
- **AND** 用户点击关闭按钮
- **THEN** 系统关闭当前插件页面
- **THEN** 系统清空搜索词
- **THEN** 内容主体恢复展示工具主页默认内容
- **THEN** 顶部状态区域恢复展示搜索输入区

#### Scenario: 插件标题区接入前端基座

- **WHEN** 顶部状态区域展示插件标题区
- **THEN** 关闭按钮的可访问名称等用户可见文案来自应用级 i18n message
- **THEN** 插件标题区位于现有 Naive UI Provider、应用主题、Naive UI locale/dateLocale 同步机制之下
- **THEN** 插件标题区在 light 与 dark 主题下保持可读
